Fields and Properties

Understanding Fields and Properties

  • Kotlin properties provide a convenient way to define mutable (var) or immutable (val) variables with implicit accessors.
  • Kotlin properties are backed by a field, referred to as field, which can be accessed inside custom getters and setters.
  • Private setters can control the mutability of a property externally.

Examples of Properties

        
class Person {
    val firstname: String = "Celine" // immutable property
    var lastname: String = "Heldner" // mutable property
}

val person = Person()
println(person.firstname)      // Has a getter only
person.lastname = "NewName"    // Has both a getter and setter

This demonstrates implicit getters and setters for Kotlin properties, highlighting the differences between mutable and immutable properties.
