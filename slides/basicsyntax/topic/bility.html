Nullability / Mutability / Immutability

Nullability refers to whether a variable can hold a null value. In Kotlin, for example, you can define a variable as nullable by adding a question mark after the type:

        
var name: String? = null // name can be null

This is important for preventing null pointer exceptions, as Kotlin forces you to handle null cases explicitly.

Mutability describes whether a variable can be changed after it is created. In Kotlin, you can declare a mutable variable using var:

        
var age: Int = 25
age = 30 // age can be changed

On the other hand, Immutability refers to variables that cannot be changed after they are initialized. In Kotlin, you declare an immutable variable using val:

        
val birthYear: Int = 1998
// birthYear = 2000 // This will result in a compilation error

Using immutability can lead to safer and more predictable code, especially in concurrent programming scenarios.
