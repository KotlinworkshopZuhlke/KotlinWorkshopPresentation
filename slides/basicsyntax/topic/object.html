Singleton

        
object Payroll {
    val allUsers = arrayListOf<User>()

    fun calculateTotalSalaryOverUsers() {
        for (user in allUsers) {...}
    }
}

In Kotlin, an object declaration creates a singleton, which means there's only one instance of that class. This is useful for managing shared state or behavior without needing to create multiple instances.

object Payroll: Defines a singleton named Payroll. allUsers: A mutable list that can store multiple User instances. calculateTotalSalaryOverUsers: A function that iterates through allUsers to perform salary calculations.

Anonymus inner class

        
object IgnoreCaseFileComparator : Comparator<File> {
    override fun compare(file1: File, file2: File): Int {
        return file1.path.compareTo(file2.path, ignoreCase = true)
    }
}

val files = listOf(File("/Z"), File("/a"))
println(files.sortedWith(IgnoreCaseFileComparator))

An anonymous inner class is a class without a name that is defined at the point of instantiation. It is often used to implement interfaces or extend classes on the fly.

object IgnoreCaseFileComparator: An anonymous inner class implementing Comparator<File>. compare function: Provides a way to compare two File objects by their paths, ignoring case differences. sortedWith: Sorts a list of File objects using the custom comparator defined above.

Companion Object

        
class A {
    companion object {
        fun bar() {
            println("Companion object called")
         }
     }
}

fun main() {
    A.bar()
}

Companion objects are similar to static members in Java. They allow you to define methods and properties that belong to the class rather than to instances of the class.

companion object: Defines a companion object within class A. bar function: A static-like method that can be called without an instance of A.
